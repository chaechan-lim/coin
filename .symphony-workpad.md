### Bug Analysis
**Root cause**: `sync_exchange_positions()` lines 1090-1102 clear the DB position (quantity=0) and record order/PnL, but for futures the cash handling block (lines 1109-1127) only does audit logging. The margin (invested) + PnL is never returned to `self._cash_balance`.

**Fix**: Added cash return logic in the position clearing loop. When `is_futures` and a position is cleared, add `max(invested + pnl_amount, 0)` back to `self._cash_balance` and update `self._realized_pnl`. The `max(..., 0)` prevents negative cash return on forced liquidation (where loss > margin).
